Application Lifecycle Management (ALM) Toolkit Simplifies Version Control and Deployment in Power BI

ALM Toolkit improves version control and deployment for Power BI data by letting users selectively push or update models onto target environments.

Power BI
Share
Tweet

Business Case:

When building enterprise applications, DevOps teams manage separate environments so that end user impact is minimized during feature releases. Maintaining separate environments allows teams to control which features are pushed to production.

When moving Power BI reports and data models from one environment to another, DevOps teams need to know the differences in measure calculations and dimensions between the source and target environments. Comparing changes in the environments becomes even more complex when multiple users are working on feature updates. Currently, there is no solution easily monitor changes in Power BI production environments and to merge work done by multiple users in Power BI.

To improve version control and deployment in Power BI, we partnered with the Microsoft Power BI team to develop the Application Lifecycle Management (ALM) Toolkit.

Key Challenges:
  • Ability to compare data models between two Power BI reports.
  • Allow multiple users to work on the same data model and reduce the effort required to merge changes.
  • Ability to selectively deploy changes to a target environment.
  • Create a report detailing changes that can be easily shared with stakeholders.

Solution:

ALM Toolkit allows users to compare two data models, generate a report detailing changes, and selectively push changes to production.

Key Highlights:
  • View differences between two data models.
  • Selectively push changes by allowing users to skip, update, create, or delete prior to making an update.
  • Generate an Excel report with the differences between two data models.
  • Update the target model from directly within the toolkit.
Figure 1: Connections selection prompt

Users begin by selecting the source and target workspace and dataset (Figure 1). The differences window is then loaded along with a diff editor that allows users to see which definitions were updated (Figure 2).

Figure 2: Differences screen

The contextual menu allows users to skip multiple items (Figure 3). Alternatively, users can click the notch next to any cell to select individual rows.

Figure 3: Navigation customization

If a user does not select a parent node for objects that are to be created, corresponding children will not be created. Similarly, if a user deletes a parent node, the corresponding children will be deleted.

Figure 4: Custom report creation

Once a user validates his selections using the validate button, he can use the update button to update the target. Users can also export a list of the changes into Excel using the Report Differences button. Users can save the comparison and selections and later open the saved file if they do not want to update the target immediately.

Business Outcome:

ALM Toolkit improves version control and deployment of Power BI reports, enabling users to efficiently track changes, push features to production, merge changes from multiple users, and create reports detailing all changes.

Outcome Highlights:
  • Improved productivity and reduced cost by reducing labor required to merge data models.
  • Improved end-user experience with the ability to control changes that are pushed to a target environment.
  • Improved efficiency by enabling users to generate reports of changes to share with stakeholders.
